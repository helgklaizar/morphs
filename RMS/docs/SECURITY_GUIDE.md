# 🛡️ RMS AI OS: Security & Auditing Guide

Этот документ описывает, как проводить аудит безопасности проекта и защищать его от взлома.

## 🛠️ 1. Методы Тестирования (Как проводить тесты?)

### А) Автоматизированные тесты (SAST/DAST)
1. **Dependency Audit**:
   ```bash
   pnpm audit
   ```
   *Что ищет*: Уязвимости в сторонних библиотеках (Node.js).
2. **Gitleaks (Секреты)**:
   Скачайте `gitleaks`. Команда: `gitleaks detect -v`.
   *Что ищет*: Хардкод паролей, API-ключей и токенов в истории Git.
3. **Rust Audit (Tauri)**:
   `cargo audit` в директории `src-tauri`.

### Б) Ручное тестирование (Penetration Test)
1. **Проверка API Rule (PocketBase)**:
   - Откройте PocketBase Admin (`https://rms.shop/_/`).
   - Перейдите в настройки коллекции (например, `orders`).
   - Проверьте вкладку "API Rules".
   - **ТЕСТ**: Попробуйте сделать GET запрос к `collections/orders/records` без авторизации (например, через `curl`). Если данные вернулись — база открыта для всех! Должно быть `id = @request.auth.id` или аналогичное.
2. **Проверка Admin Backend**:
   - Попробуйте зайти в админку с неправильным паролем. 
   - Проверьте, нет ли в `src-tauri` файла `sync.rs` с захардкоженным паролем (это было найдено в предыдущем аудите!).
3. **Проверка Telegram Бота**:
   - Если бот принимает команды (`/order`), проверьте, есть ли там проверка `chat_id` администратора. Если нет — любой человек может спамить командами.

## 🚨 2. Список Критических Уязвимостей (Checklist)

| Уязвимость | Статус | Как исправить |
| :--- | :---: | :--- |
| **Хардкод пароля в Rust (Tauri)** | 🔴 КРИТИЧНО | Удалить пароль из `sync.rs`. Использовать `Tauri Storage` или требовать вход юзера. |
| **Открытые API Rules в PocketBase** | 🟡 СРЕДНЕ | Проверить, что коллекции `orders` и `inventory` закрыты для анонимных GET/POST. |
| **Отсутствие Rate Limiting** | ⚪ НИЗКО | Настроить Nginx/Caddy, чтобы ограничить количество запросов в секунду (от DDoS). |
| **Слив API ключей в Git** | 🔴 КРИТИЧНО | Использовать `.env` и добавить его в `.gitignore`. Никогда не пушить ключи! |

## 🔍 3. Инструменты для Глубокого Аудита

1. **Burp Suite / OWASP ZAP**: Прокси-серверы для перехвата и изменения HTTP-запросов. Можно "подменить" цену товара в корзине перед отправкой на сервер.
2. **Postman**: Для ручного тестирования всех эндпоинтов API PocketBase.
3. **Nmap**: Проверить открытые порты на VPS (должны быть только 80, 443 и 22).

---

## 🧼 4. Безопасность Базы Данных (PocketBase)

PocketBase — это SQLite. Поэтому:
- **Удаленные бэкапы**: Скрипт `backup.sh` на сервере должен копировать `data.db` в безопасное место, НЕ доступное через веб.
- **Права доступа**: Файл `data.db` должен иметь права `600` (только для владельца процесса PocketBase).
