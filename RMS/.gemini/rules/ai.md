# ПРАВИЛА ДЛЯ AI (ОБЯЗАТЕЛЬНО К ИСПОЛНЕНИЮ)
1. **ПУШИТЬ ВСЕ ИЗМЕНЕНИЯ:** После КАЖДОГО внесенного изменения и рабочего шага обязательно делать `git add . && git commit -m "..." && git push` для сохранения истории.
2. **КРАТКИЕ ОТВЕТЫ:** Всегда отвечай максимально кратко, по делу и строго на заданный вопрос без лишней воды.
3. **МАРКИРОВАТЬ ТЕХДОЛГ:** Любой временный код, костыль или известный баг ОБЯЗАН быть помечен стандартным комментарием:
   - `// TODO: описание` — нужно сделать позже
   - `// FIXME: описание` — сломан, нужно починить
   - `// HACK: описание` — костыль, требует рефакторинга
   - `// XXX: описание` — опасный участок, требует особого внимания
   Без маркировки техдолг невидим и накапливается незаметно.
