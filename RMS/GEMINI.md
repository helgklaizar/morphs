## 🚀 BUILD & DEPLOY
1. Проверить версию в `src-tauri/tauri.conf.json`.
2. Команды (в монорепе):
   ```bash
   pnpm run build && pnpm run tauri build
   ```
3. **Troubleshooting:** Если билд падает с ошибками Turbopack/Next.js, почистить кэш:
   ```bash
   rm -rf apps/backoffice/.next
   ```
4. После билда бинарник `.app` появляется в `apps/backoffice/src-tauri/target/release/bundle/macos/`.
5. Копировать в `/Applications`:
   ```bash
   cp -r "apps/backoffice/src-tauri/target/release/bundle/macos/Backoffice New.app" /Applications/
   ```

## 🌍 ИНФРАСТРУКТУРА И АРХИТЕКТУРА (Local vs VPS)
- **Монорепа**: Состоит из `apps/backoffice` (Десктоп приложение Tauri/Next), `apps/landing` (Публичный веб), `packages/types`, `packages/db`.
- **База Данных**: PocketBase (`sqlite`). Бек-офис общается с БД через URL из переменной окружения `NEXT_PUBLIC_PB_URL`.
- **Локальный Dev**: Лендинг запущен локально `pnpm run dev --filter landing`.
- **Синхронизация**: Приложение Backoffice использует репозитории (как `OrdersRepository`) для двустороннего обмена JSON.
- **VPS Прод Сервер**: 
  - Находится в Google Cloud. Подключение и пуш файлов: `gcloud compute scp`.
  - Деплой: Сборка Next.js `standalone` тарболла `.tar.gz`, выгрузка на сервер, распаковка и `pm2 restart <имя_приложения>`.
- **AI Интеграция**: Настройки ИИ переключаемые (Cloud vs Local). Поддерживается Ollama (`http://localhost:11434`) для запуска локальных LLM (Gemma 4b) на Apple Silicon для приватности.

- [x] **Offline-Fallback для Меню:** Приложение больше не зависит от интернет-подключения для отображения витрины блюд. Поддерживается ограниченное локальное добавление блюд, переключение видимости и изменение стоков через Outbox.
- [x] **Offline-Fallback для Столов (Tables):** Карта зала переведена на локальную БД SQLite, что позволяет официантам сажать гостей и администраторам редактировать карту зала автономно.
- [x] **Offline-Fallback для Рецептов и Сборок:** Создание и редактирование ТТК работает оффлайн. `SyncEngine` синхронизирует рецепты + ингредиенты / сборки + позиции пакетами через Outbox.
- [x] **Offline-Fallback для Поставщиков и Закупок:** Поставщики кэшируются в SQLite при старте с сетью; при отключении — используется кэш. Заказы поставщикам создаются и меняют статус через Outbox; `SyncEngine` доставляет их на сервер при реконнекте.
