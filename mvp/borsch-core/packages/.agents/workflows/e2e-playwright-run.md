---
description: Воркфлоу для координации с UI и запуска сквозного тестирования E2E (Playwright) для проектов со сложными анимациями (Framer Motion).
---

0. **СИСТЕМНОЕ:** Строго первым делом немедленно выведи в чат: `🔄 **Контекст:** [Агент: QA/Tester] | [Скилл: None] | [Воркфлоу: /e2e-playwright-run]`

1. **Запуск Окружения:**
   - Подними локальный дев-сервер: `npm run dev &`.
   - Дождись ответа HTTP-портов.
2. **Подготовка Playwright:**
   - Если E2E требуют БД, накати seed'ы тестовых данных.
   - Настрой глобальные моки: например, отключи `window.matchMedia` или переопредели reduced_motion, если анимации мешают стабильности тестов, либо наоборот — включи их, чтобы проверить "Framer Motion layout shifts".
3. **Запуск Test Runner:**
   - Запусти тесты: `npx playwright test`.
   - Опционально используй `--ui` или собирай `trace` файлы для отладки падающих тестов `npx playwright test --trace on`.
4. **Анализ Результатов (Ошибки анимаций):**
   - Если тест падает из-за таймаута при поиске элемента (ElementNotInteractable), проанализируй, не заблокирован ли он анимацией выхода (Exit Animation). 
   - Предложи патч, внедряющий waitFor или исправляющий `mode="wait"` в `AnimatePresence`.
5. **Теардаун:** Убей `npm run dev` порты.
6. Составь отчет с результатами E2E (покрытые маршруты) и прикрепи логи Playwright.
