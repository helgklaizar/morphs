# Git Hooks Agent Gate

**Когда использовать:** При переходе задачи от одного ИИ (Engineer) к другому (QA или Auditor) или перед финальным 'Сохранением состояния' в `GEMINI.md`. Разделение ответственности как в игровых студиях.

## Правила работы ИИ:
1. **Никогда не пушь сломанный код.**
2. Я (как ИИ-оркестратор) использую локальные pre-commit/pre-push хуки в проекте как **Quality Gate**.
3. Перед финальным сохранением кода, я должен запустить локальные линтеры (`npm run lint`, `cargo clippy`, `pytest` и т.д.).
4. Если хук возвращает ошибку — код НЕ ПЕРЕДАЕТСЯ заказчику. Я должен самостоятельно:
   - Открыть ошибки;
   - Переключиться в режим "Debugger";
   - Исправить их ДО того, как скажу пользователю "Я всё сделал".
5. У каждого агента есть своя зона (Frontend не ломает Backend файлы). Координация идет через коммиты и жесткие пайплайны.
