# RMS AI OS — Project Memory (Borsch Core MVP)

> **Статус**: Минимальный рабочий прототип (MVP) выделен в отдельный проект.
> **Расположение**: `/Users/klai/AI/mvp/borsch-core` 

## ⚙️ Локальный контекст (Стек и Архитектура)
- **Архитектура:** Monorepo (Turborepo + pnpm)
- **Стек Фронта:** Next.js 14+ (App Router), TailwindCSS, Zustand
- **Стек Бэка:** PocketBase (Hosted on VPS: `https://borsch.shop`)
- **Tauri / Оффлайн:** **Отключен**. Мы полностью отказались от старого механизма локальной SQLite+SyncEngine. Прямые `pb.collection` запросы к VPS.
- **Telegram:** Полностью работает на серверной стороне (VPS, файл `pb_hooks`). Фронтенд туда не лезет. 

## 📦 Структура Проекта
1. `apps/backoffice/` (порт 3001) — Админ-панель:
   - **Оставлено:** `/pos` (Касса), `/orders` (Лента Заказов), `/menu` (Меню), `/dashboard` (Легкий Обзор). 
   - **Вырезано/Удалено:** Финансы, Списание, Склады, Рецепты, Лояльность, Клиенты/CRM, AI Sidebar, AI ассистент. 
2. `apps/landing/` (порт 3000) — Лендинг для клиентов (доставки/самовывоз).
3. `packages/core/` — Переиспользуемая бизнес-логика (Stores). Оставлены только нужные: `useMenuStore`, `useOrdersStore`, `useAuthStore`.

## 🚀 Как запускать
```bash
# Из корня /Users/klai/AI/mvp/borsch-core
pnpm dev
# Поднимет landing на localhost:3000
# Поднимет backoffice на localhost:3001
```

## 🌍 Важные нюансы
- В `apps/backoffice/.env` лежит переменная соединения с БД:
  `NEXT_PUBLIC_PB_URL=https://borsch.shop`
- **Проблема с датами решена**: Использована функция `safeFormat` для дат `reservationDate / createdAt`, которые приходят из старой БД строками.
