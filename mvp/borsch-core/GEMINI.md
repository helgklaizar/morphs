# RMS AI OS — Project Memory (Borsch Core MVP)

> **Статус**: Минимальный рабочий прототип (MVP) выделен в отдельный проект.
> **Расположение**: `/Users/klai/AI/mvp/borsch-core` 

## ⚙️ Локальный контекст (Стек и Архитектура)
- **Архитектура:** Monorepo (Turborepo + pnpm)
- **Стек Фронта:** `apps/backoffice-vite` (Vite, React Router, TailwindCSS), `apps/landing` (Next.js 14+)
- **Стек Бэка:** Hono + Prisma + SQLite (`apps/api` на порту 3002).
- **Миграция:** Проект в процессе перехода с PocketBase на собственную API инфраструктуру (Hono).
- **Tauri / Оффлайн:** Оффлайн отключен. Десктопное приложение — просто оболочка.
- **Интеграция:** Реалтайм реализован через SSE (`/api/events`).

## 📦 Структура Проекта
1. `apps/backoffice-vite/` (порт 3001) — Новая админ-панель на Vite:
   - **Активные Модули:** 
     - `/pos` (Касса), `/orders` (Лента Заказов с интегрированной вкладкой `ClientsTab`), `/menu` (Меню).
     - **Склад и Закупки (Восстановлены и стандартизированы):** `/inventory` (Склад с категориями без Drag&Drop, с жесткой привязкой к кнопкам), `/recipes` (Умные рецепты с автоуказанием единиц измерения), `/purchases` (Закупки авто-считающие недостаток склада по активным заказам).
   - **Вырезано/Удалено:** Старая громоздкая CRM, Финансы, Списание, AI Sidebar, AI ассистент (заменены на минималистичные интеграции). 
2. `apps/landing/` (порт 3000) — Лендинг для клиентов (доставки/самовывоз).
3. `packages/core/` — Переиспользуемая бизнес-логика и API клиент. Полностью переписана под FSD (домены: `orders`, `menu`, `inventory`, `clients`, `cart`).
4. `apps/backoffice/src-tauri` — Десктопная обертка на базе Tauri 2.0 (сохранение геометрии окна через `window-state:default` plugin).

## 🌍 Важные нюансы
- В `apps/backoffice/.env` лежит переменная соединения с БД: `NEXT_PUBLIC_PB_URL=https://borsch.shop`
- **Проблема с датами:** Функция `safeFormat` для дат `reservationDate / createdAt` решает проблему строковых дат старой БД.
- **Tauri Оффлайн отключен:** Но само Tauri приложение (macOS) собирается для обертки (`/deploy-desktop`).
- **Генерация Закупок:** Черновики заказов генерируются с учетом дефицита `(Заказы * Порции) - Склад`. Учитываются единицы измерения (`кг`, `упаковка`).
