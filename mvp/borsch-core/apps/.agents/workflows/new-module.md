# /scaffold-module

Цель: правильно добавить новый модуль в существующий проект. Отличается от `/new-project` — работает внутри уже существующей архитектуры.

## Входные параметры
- `--project <name>` — проект (например, `RMS`, `TeleFeed`)
- `--module <name>` — имя модуля (например, `inventory`, `notifications`)
- `--type <feature|service|page|api>` — тип модуля

## Шаги

### 1. Погружение в контекст
1. Прочитать `<project>/GEMINI.md` — стек, архитектура, ключевые решения.
2. Прочитать `<project>/docs/architecture.md` — понять как устроены существующие модули.
3. Найти пример похожего модуля в кодовой базе — взять за образец структуру.

### 2. Дизайн модуля
Подготовить краткий план (без кода) и показать пользователю:
```
Модуль: <name>
Тип: <type>
Файлы которые создадим:
  - src/<module>/index.ts
  - src/<module>/types.ts
  - src/<module>/store.ts (если stateful)
  - src/<module>/__tests__/
  - docs/<module>.md
API-контракт (если type=api):
  - POST /api/<module>/
  - GET /api/<module>/:id
```
**Стоп: дождаться одобрения пользователя.**

### 3. Создать файловую структуру
По стандарту проекта:
- **Next.js / React**: `src/features/<module>/` (компоненты, хуки, сторы, типы)
- **Python**: `<project>/src/<module>/` (`__init__.py`, `models.py`, `service.py`, `router.py`)
- **Rust/Tauri**: `src-tauri/src/<module>/` (`mod.rs`, `commands.rs`, `state.rs`)

### 4. Создать тесты (TDD-first)
- Написать тесты до реализации (скелет с `todo!()`/`expect.todo()`).
- Убедиться что тесты падают с правильной ошибкой.

### 5. Реализовать модуль
- Следовать паттернам из `<project>/GEMINI.md → ## Ключевые решения`.
- Не нарушать constraints из `<project>/.gemini/rules/constraints.md`.

### 6. Обновить документацию
- `docs/architecture.md` — добавить модуль в схему Mermaid.
- `docs/api.md` — добавить новые эндпоинты.
- `TODO.md` — закрыть или добавить следующие задачи по модулю.

### 7. Прогнать тесты
```bash
# Убедиться что всё зелёное
npm run test / pytest / cargo test
```

### 8. Обновить GEMINI.md проекта
- Секция `## Что делали последним`.
- Секция `## Следующие задачи` — убрать выполненное.
