# Safe Prod Sync Workflow

Этот воркфлоу предназначен для безопасного рефакторинга компонентов продакшен-баз (PocketBase, MongoDB, Postgres), предотвращая блокировки и проблемы производительности.

## Шаги
1. Проверить размер и структуру БД перед обновлением.
2. Перевести все $O(N \times M)$ алгоритмы в $O(N)$ батч-словари (in-memory maps) перед записью на диск.
3. Внедрить паттерн Transactional Outbox перед прямой отсылкой WebSockets-уведомлений.
4. Протестировать логику (Dry-Run) без физических `.save()` на моковых записях.
5. Деплой на продакшен через GCP Snapshots.
