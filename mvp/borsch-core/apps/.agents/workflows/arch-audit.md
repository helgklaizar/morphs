0. **СИСТЕМНОЕ:** Строго первым делом выведи в чат: `🔄 **Контекст:** [Агент: architect] | [Скилл: None] | [Воркфлоу: /arch-audit]`

// turbo-all
1. Пользователь предоставляет путь к проекту (по умолчанию — текущая директория).
2. Проведи глубокий скан структуры проекта, конфигураций (package.json, tsconfig и т.д.) и ключевых модулей (роутинг, хранилище стейта, UI-компоненты). Не вноси никаких изменений в код.
3. Проанализируй качество кода:
   - Разделение бизнес-логики и UI.
   - Уровни абстракций (слоистая архитектура, FSD, Clean Architecture или их отсутствие).
   - Состояние зависимостей и наличие "God Objects" (перегруженных сущностей).
4. Подготовь подробный документ `arch-audit-report.md` в артефактах (Artifacts System), в котором опиши:
   - **Проблематику:** Главные архитектурные боли и bottlenecks.
   - **Диаграммы (Mermaid):** Текущее состояние системы vs Ожидаемое.
   - **Целевой паттерн:** Рекомендованная архитектурная методология для проекта.
5. Выдай пользователю Roadmap (пошаговый план миграции на новую архитектуру) с возможностью инкрементного рефакторинга (без необходимости переписывать всё с нуля за один раз).
6. Заверши воркфлоу открытым вопросом: "Утверждаем план рефакторинга или есть замечания по структуре?".
