0. **СИСТЕМНОЕ:** Строго первым делом выведи в чат: `🔄 **Контекст:** [Агент: architect] | [Скилл: migration] | [Воркфлоу: /tech-evolution]`

// turbo-all
1. Пользователь вызывает команду в целевом проекте (или указывает путь).
2. Проведи глубокий технический аудит проекта (просканируй зависимости через `package.json` / `Cargo.toml` / `pyproject.toml` и структуру папок). Пойми бизнес-задачу исходного репозитория.
3. Включи режим **"Радикального Архитектора"**. Задай себе вопрос: "А правильный ли вообще стек и архитектура тут используются?".
   - Предложи кардинально лучшее решение (Например: переход с Vanilla React на Astro/Next.js; с монолита на Clean Architecture/Hexagonal; с REST на tRPC и т.д.).
   - Опиши, какую методологию (FSD, DDD, TCA) лучше применить и *почему*.
4. Создай артефакт `implementation_plan.md`, в котором опиши:
   - Текущие проблемы стека/архитектуры.
   - Предлагаемый новый стек и архитектурный паттерн.
   - Roadmap жесткой структурной миграции (какие папки создать, какие зависимости обновить, как переписать файлы).
5. Заверши первый этап открытым вопросом: *"Утверждаем эволюцию стека/архитектуры? Если да, я начну молча переписывать файлы и перестраивать директории по пунктам."*
6. Если пользователь дает "зеленый свет" (пишет "го", "да", "выполняй") — **АВТОМАТИЧЕСКИ** переходи к фазе Execution:
   - Реструктуризируй папки (через `mkdir`, `mv`).
   - Измени конфигурацию зависимостей.
   - Проведи рефакторинг компонентов под новый подход.
   - Трекай процесс через артефакт `task.md`.
