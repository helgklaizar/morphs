# Трекер Внешних Источников (AI Radar Data Collector)

**ТРИГГЕР:** Этот workflow срабатывает АВТОМАТИЧЕСКИ всегда, когда пользователь просит проанализировать чужой репозиторий, статью или документацию, И мы извлекаем из нее полезный контекст/код, который потом используется в проектах `Antigravity IDE`.

## Инструкция для ИИ
Если триггер сработал (или пользователь явно попросил добавить ссылки в список):

1. **Оцени ресурс**:
   - `usefulness` (1-10): насколько это реально нужно нам по сравнению с текущим стеком.
   - `our_use_case`: опиши словами как этот инструмент встроится в нашу архитектуру или заменит наш текущий флоу.
   - `category`: `Repo` / `Article` / `Concept` / `Prompt`.
   - `tags`: Ключевые слова через запятую.

2. **Запиши в Базу Данных (AI Radar)**:
   Без спроса пользователя, сгенерируй JSON-строку и вызови локальный скрипт:
   ```bash
   python ~/.gemini/tools/ai-radar/radar.py add '{"url": "...", "title": "...", "tags": "...", "category": "Repo", "usefulness": 8, "our_use_case": "Поможет с..."}'
   ```
   *(Используй метод `run_command` для выполнения этого баш-скрипта)*.

3. Если ссылок много (например, парсинг списка из телеграма), сделай цикл (loop) в bash или вызови скрипт для каждого объекта.

4. В текстовом ответе коротко упомяни: *"Я автоматически сохранил эту ссылку в `ai-radar.db`"* и укажи сколько баллов полезности ты ей дал.
