---
description: Эмуляция генеративного ИИ для анализа страниц и выдачи советов
---

0. **СИСТЕМНОЕ:** Строго первым делом немедленно выведи в чат: `🔄 **Контекст:** [Агент: Current] | [Скилл: None] | [Воркфлоу: /borsh-anal]`

Этот воркфлоу позволяет Antigravity (тебе) выступить в роли умного бизнес-ассистента и напрямую проанализировать исходники или данные конкретной страницы проекта, чтобы сгенерировать полезные советы и отобразить их в боковом чате (AI Advisor).

1. **Определить скоуп:** Узнать у пользователя целевой модуль или страницу для анализа (например, `/loyalty`, `/inventory`, `/tables`).
2. **Анализ кода:** С помощью инструмента `view_file` проанализировать `page.tsx` модуля и связанные UI-компоненты.
3. **Анализ состояния:** С помощью `view_file` изучить соответствующие Zustand сторы (например, `useLoyaltyStore.ts`, `useInventoryStore.ts`), чтобы понять, какие данные и метрики доступны на клиенте.
4. **Генерация бизнес-инсайтов:** Как AI, придумать 2–4 глубоких аналитических совета (советы не должны быть банальными). Они должны касаться оптимизации процессов, "бутылочных горлышек" или потенциала увеличения прибыли.
5. **Форматирование:** Сконвертировать эти инсайты в формат JSON по интерфейсу `AiInsight`:
   ```json
   [
     {
       "id": "insight-1",
       "type": "tip", // может быть tip, warning, success, action
       "text": "Глубокий анализ: ..."
     }
   ]
   ```
6. **Внедрение в код:** С помощью `replace_file_content` или `multi_replace_file_content` внести изменения в файл `apps/backoffice/src/store/useAiInsightsStore.ts`. Добавить сгенерированные советы прямо в fallback-моковую логику `catch` (или в ветвление `switch(module)`), заменяя стандартные значения.
7. **Safe-Commit:** Сделать обязательный коммит (в соответствии с правилами `GEMINI.md`): `git add . && git commit -m "AI: Внедрена эмуляция советов для модуля [module]"`.
8. **Отчет:** Сообщить пользователю в чат, что советы загружены и он может открыть вкладку "Советы" в боковом AI-окне.
