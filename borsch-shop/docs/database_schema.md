# Схема Базы Данных (Database Schema)

Приложение `backoffice` работает полностью offline благодаря встроенной SQLite-базе данных (используется плагин `@tauri-apps/plugin-sql`).
Этот документ подробно описывает архитектуру таблиц, генерацию ID и механизмы интеграции с облаком PocketBase.

## 1. Фундаментальные паттерны таблиц

Все таблицы в БД следуют строгим правилам:
1. **Primary Key `id`**: **ВСЕГДА 15 символов** `[a-z0-9]`. Использовать `generateId()` из `src/lib/generateId.ts`. UUIDv4 использовать **запрещено**, так как PocketBase не принимает их при сихронизации.
2. **Столбец `sync_status`**: Текстовое поле, гарантирующее Offline-First механизм.
   - `'synced'` — запись полностью совпадает с сервером.
   - `'pending_insert'` — запись создана оффлайн и ожидает отправки (POST).
   - `'pending_update'` — запись изменена оффлайн и ожидает обновления (PATCH).
3. **Отсутствие Foreign Key (FK) Constraints**: В чистом виде `FOREIGN KEY` не настраивается. Это сделано намеренно для обеспечения консистентности "слабых связей" между SQLite и NoSQL-подобным поведением PocketBase. Разрешение связей происходит на уровне бизнес-логики (Zustand).
4. **Хранение JSON в TEXT**: Сложные объекты, такие как массив цен у поставщика, хранятся как сериализованный JSON в полях с типом `TEXT` (например `prices` или `items`).

## 2. Ключевые Таблицы (Core Schema)

### Инвентарь на Складе (Сырье)
*Товары, из которых мы готовим или собираем конечные продукты.*

**Таблица `inventory_items`**
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | TEXT PK | 15 символов [a-z0-9] |
| `name` | TEXT | Название (Кефир, Ланчбокс) |
| `category_id` | TEXT | Ссылка на `inventory_categories.id` |
| `unit` | TEXT | Ед. приема товара (кг, уп, л) |
| `price` | REAL | Стоимость одной `unit` |
| `pack_size` | REAL | Информационное поле размера пачки |
| `sync_status` | TEXT | Статус синхронизации ('synced' и др.) |

**Таблица `inventory_categories`**
Определяет доступность: Флаг `is_visible_in_recipe` (Показывать при готовке) или `is_visible_in_assemblies` (Показывать при упаковке). Разделяет мясо и картон.

---

### Производство (Рецепты и Сборки)
*Сущности, которые агрегируют `inventory_items` и конвертируют сырье в блюдо или упаковку.*

**Таблицы `recipes` и `assemblies`**
- Хранят `id` и `name`. 

**Таблицы связей `recipe_ingredients` и `assembly_items`**
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | TEXT PK | Уникальный ID связи |
| `recipe_id` (или `assembly_id`) | TEXT | К какому Рецепту/Сборке относится |
| `inventory_item_id` | TEXT | Какой Ингредиент используется |
| `quantity` | REAL | Сколько ингредиента заложено (в `recipeUnit`) |

---

### Витрина (Меню)
*То, что видит конечный клиент на лендинге Next.js.*

**Таблица `menu_items`**
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | TEXT PK | |
| `name`, `name_en`, ... | TEXT | Локализованные названия |
| `price` | REAL | Цена для клиента (Client Price) |
| `description`... | TEXT | |
| `image_url` | TEXT | Ссылка на картинку в облаке PocketBase Storage |
| `recipe_id` | TEXT | Связанный Recipe (Для себестоимости) |
| `assembly_id` | TEXT | Связанная Упаковка (Для себестоимости) |
| `category_id` | TEXT | Ссылка на `menu_categories.id` (Супы, Напитки) |

---

### CRM (Заказы клиентов)
*Поступают в систему либо руками кассира, либо автоматически скачиваются SyncEngine из PocketBase (от Landing-сборки).*

**Таблица `orders`**
- `status`: 'new', 'accepted', 'ready', 'delivered', 'cancelled'.
- `total_amount`: REAL
- `customer_name`, `customer_phone`: TEXT
- `payment_method`: 'cash', 'card', 'transfer'

**Таблица `order_items`**
Детализация конкретного заказа (Что заказали). 
- `order_id`: Ссылка на заказ.
- `menu_item_name`: Записываем сюда строку, а не только `menu_item_id`, чтобы чек клиента не сломался, если мы потом удалим блюдо из меню.
- `price_at_time`: Фиксация цены на момент заказа.

---

### Закупки (Suppliers & B2B)
*Модуль взаимодействия с контрагентами (поставщики мяса, упаковки).*

**Таблица `suppliers`** — Карточки поставщиков (Имя, Телефон, Адрес).
**Таблица `supplier_products`**
- `prices`: TEXT (JSON) — Прайс-лист на товар (Например: `{"шт": 100, "уп": 500}`). 
- Запись сериализуется из JS объекта перед сохранением в `localDb.ts`.
**Таблица `supplier_orders`** — Зарезервировано под формирование Корзины закупок и отправки чеков в WhatsApp/Email поставщикам. Поле `items` хранится как JSON.

---

### Настройки и Прочее
- **`app_settings` / `landing_settings` / `ai_settings`**: Таблицы-синглтоны. Обычно в них всегда ровно одна запись. Хранят лендинг-сетап, токены для AI и лимиты.
- **`clients`**: CRM лояльных клиентов.
- **`translations`**: Локальный кэш словарей локализации (RU/EN/HE/UK) для мгновенного переключения языка админки и сайта.
