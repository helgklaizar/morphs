# Документация проекта Borsch Shop

Эта папка содержит архитектурные схемы, API-манифесты и историю спринтов разработки.

## Структура

- `api.md` - Контракты API и детали интеграции (PocketBase, Telegram).
- `architecture.md` - Формализованный граф архитектуры (в формате Mermaid), визуально объясняющий взаимосвязи компонентов Backoffice, облака PocketBase и Web UI.
- `architecture_manifesto.md` - Развернутый философский и технический манифест. Почему мы используем SQLite, Zustand и Offline-First подход. Как мы обходим ограничения PocketBase (например, 15 символьные ID вместо UUIDv4).

### Спринты (sprints/)

- `sprints/week-1.md` - История первых тасок, задач на неделю 1.
- `sprints/week-2.md` - Задачи второй недели спринта.

Любые крупные архитектурные изменения должны быть сначала задокументированы здесь.
